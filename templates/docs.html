How MosPay Works

Architecture

MosPay operates on a microservices architecture with the following components:

  • API Gateway: Handles authentication and request routing
  • Payment Processor: Processes payment requests through PostgreSQL functions
  • Transaction Manager: Tracks and manages transaction states
  • Notification System: Sends real-time updates via webhooks

Payment Flow

  1. Client authenticates and receives JWT token
  2. Payment request is submitted with transaction details
  3. MosPay processes payment through microservices
  4. Transaction status is updated in real-time
  5. Webhook notifications are sent to client's callback URL

Authentication

MosPay uses a two-step authentication process for maximum security:

POST /api/v1/auth/login

Authenticate with your API credentials to receive a JWT token.

Headers: Authorization: Basic {base64(username:password)}

Step 1: Get Access Token

curl -X POST "https://api.mospay.com/api/v1/auth/login" \
  -H "Authorization: Basic $(echo -n 'your_username:your_password' | base64)" \
  -H "Content-Type: application/json"

Step 2: Use JWT Token

Include the JWT token in the Authorization header for all subsequent API calls:

curl -X POST "https://api.mospay.com/api/v1/payment/process" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -H "X-App-ID: YOUR_APP_ID"
Security Notes
  • Never expose your API credentials in client-side code
  • Store credentials securely on your server
  • JWT tokens expire after 24 hours
  • Always use HTTPS for API communications

API Endpoints

MosPay provides three main API endpoints for payment processing:

POST /api/v1/auth/login

Authenticate and receive JWT token for API access.

Response: JWT token with client permissions and service access.

POST /api/v1/payment/process

Process a new payment transaction.

Headers: Authorization: Bearer {jwt_token}, X-App-ID: {app_id}

POST /api/v1/payment/status

Check the status of an existing transaction.

Headers: Authorization: Bearer {jwt_token}, X-App-ID: {app_id}

Request Payload Format

All payment requests use a standardized payload format with fields f000 through f010:

Field Description Type Required
f000 Service name (e.g., "mtnmomorwa") String Yes
f001 Action type ("SERVICE" for process, "BASE" for status) String Yes
f002 Route/command name (e.g., "collection") String Yes
f003 Your App ID String Yes
f004 Transaction amount (in smallest currency unit) Number Yes
f005 Mobile number String Yes
f006 Username/account identifier String Yes
f007 Encrypted password String Yes
f008 Plain password String Yes
f009 Device/app identifier String Yes
f010 Unique transaction ID String Yes

Python Example

import requests
import base64
import json

# Configuration
API_BASE_URL = "https://api.mospay.com/api/v1"
USERNAME = "your_username"
PASSWORD = "your_password"
APP_ID = "your_app_id"

# Step 1: Authenticate
auth_string = f"{USERNAME}:{PASSWORD}"
auth_bytes = auth_string.encode('ascii')
auth_b64 = base64.b64encode(auth_bytes).decode('ascii')

auth_response = requests.post(
    f"{API_BASE_URL}/auth/login",
    headers={
        "Authorization": f"Basic {auth_b64}",
        "Content-Type": "application/json"
    }
)

if auth_response.status_code == 200:
    token = auth_response.json()["access_token"]
    
    # Step 2: Process Payment
    payment_payload = {
        "f000": "mtnmomorwa",
        "f001": "SERVICE",
        "f002": "collection",
        "f003": APP_ID,
        "f004": 5000,
        "f005": "+250788123456",
        "f006": "john_doe",
        "f007": "encrypted_pwd_123",
        "f008": "mypassword",
        "f009": "mobile_app_v1",
        "f010": "txn_20250104_001"
    }
    
    payment_response = requests.post(
        f"{API_BASE_URL}/payment/process",
        headers={
            "Authorization": f"Bearer {token}",
            "Content-Type": "application/json",
            "X-App-ID": APP_ID
        },
        json=payment_payload
    )
    
    print("Payment Response:", payment_response.json())
else:
    print("Authentication failed:", auth_response.text)
